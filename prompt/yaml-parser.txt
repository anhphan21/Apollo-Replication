Task:
Create the c++ module to parse and export the netlist and library information of PIC circuit in YAML format

Context: 
- The module is written in c++ and placed under `place_io` module
- The module includes `YamlDataBase.h`, `YamlDataWriter.h`, `YamlDataWriter.h` and `YamlDataDriver.h`
- The module creates its own database to store all information of the netlist from yaml input which is the content of `YamlDataBase`. The Database of the program is `PlaceDb` inherits the class `YamlDatabase`
- The example for the yaml file is the toy case example file under "benchmarks/PIC/**.yml"
- The format of yaml follows by structure:
	- instances: components of the circuit with `settings` section has macro_type and placement information
		- macro_type:
		- placement information:
			- Placement status: placed/fixed
			- Placement location: x/y
			- Placement orientation: N/E/S/W
			- Halo information: left/down/right/up
	- schematic_placements: location of the components in the schematic (not useful for the physical design), reserve for later but not useful now
	- nets: netlist of the circuit
		-  Format: net_name: ['component_name, port_name', ...]
	- ports: I/O of the circuit
	- settings:
		- version: version of yaml
		- design: design name
		- unit_distance_microns: convert value for the distance value
		- die_area:
			- bottom left coordinate
			- top right coordinate
		- num_instances: #instances of the design
		- num_nets: #nets of the design
		- num_ports: #port of the design
		- wg_radius: radius of waveguide (I'm not sure the meaning of this but keep the value for reference)
	- library: library information of the cell 
		- type: type of the cell
		- origin: original of the macro then we calculate the pin location from this
		- size: size of cell library
		- site: site alignment
		- pins: pin information of the cell
			- Pin offset x/y: pin offset calculate from macro origin
			- pin witdh: width of waveguide matching with the port
			- pin orientation: port orientation angle (0, 90, 180, 270)
			- pin layer: layer id for the pin
	- constraints: list of constraints apply for the design
		- constraint name
			- type: constraint type (alignment/uniform)
			- settings: setting for the constraint (for alignment: left/right/bottom/up, for uniform: vertical/horizontal)
			- objects: list of components applied the constraints
	- crossing waveguide width and height
- Create the enum in `Enum.h` to store the port orientation (0.0, 90.0, 180.0 and 270.0, which is stored as the double in the yaml input), constraint types (alignment/uniform). Adding the Yaml input format to the `FileFormat` in `Enum.h`
- Modify the `Params.h` to  handle the Yaml input and cReate variable to check if it is the yaml input file
- Modify the `Pin.h` that each pin has the variable to store the port orientation
- Modify the `PlaceDB` to handle the YamlInput and also create the callback functions (follow the logic from lef/def/bookshelf/verilog) format to parse the information of yaml netlist from parser 
- Modify the `PyPlaceDB.h` to adapt the new variable from the Yaml and create the new variables to record the number of port on each side of the node (4 sides for each node). try to follow the scheme to store value from exist variables  
- Modify the binding function `write` anbd `apply` in the `place_io` to parse the input from yaml input file
- Add the `readYaml` function in the `PyPlaceDB` to read the Yaml input file and create the python database for the input
- Modify the `PybindPlaceDB` and `PybindPyPlaceDB` to map the Python database with the c++ database

- The `YamlWriter` takes the information from the `YamlDataBase` and export the yaml output
